# 🤖 Trabalho-AI

## 📌 Sobre o Projeto

O Trabalho-AI é um sistema interativo desenvolvido por Gabriel Alves com o objetivo de explorar conceitos de inteligência artificial, experiência do usuário (UX) e desenvolvimento web.

O projeto reúne:

* Um chatbot inteligente (WannaCryBot)
* Um dashboard de comparação de IAs
* Um sistema interativo com navegação simples e intuitiva

---

# ⚖️ Comparador de IAs

## 📖 O que é

Uma funcionalidade que permite testar diferentes IAs e comparar suas respostas.

## ⚙️ Como funciona

* O usuário digita uma pergunta
* O sistema envia essa pergunta para diferentes modelos de IA
* As respostas são exibidas lado a lado

## ▶️ Como usar

1. Digite uma pergunta no campo indicado
2. Clique no botão de comparar
3. Aguarde as respostas
4. Analise as diferenças entre as IAs

## 💡 Dica

Teste perguntas como:

* Perguntas técnicas
* Perguntas criativas
* Perguntas pessoais

# 🧠 WannaCryBot

## 📖 O que é

O WannaCryBot é um chatbot desenvolvido em Python utilizando Streamlit e integração com modelos de IA via API.

## ⚙️ Como funciona

* O bot utiliza um modelo de linguagem via API
* Um sistema de "treinamento" por prompt define comportamento
* Ele adapta a forma de falar com base no usuário

## 🗣️ Comportamento

O bot foi programado para:

* Ser amigável, divertido e inteligente
* Adaptar o tom conforme o usuário
* Criar conversas naturais

## 👤 Personalização

O bot reconhece nomes e adapta o comportamento:

* **Yasmin** → tratamento mais carinhoso e atencioso
* **Amanda** → abordagem empática e acolhedora
* **Pedro** → foco em programação
* **Kauã** → apoio e motivação
* Outros usuários → comportamento adaptativo

## ▶️ Como usar

1. Acesse a área do chatbot pela Landing Page
2. Digite uma mensagem
3. Se necessário, informe seu nome
4. Continue a conversa normalmente

# 🔐 Segurança

As APIs utilizadas estão espostas no código pra testar a funcionalidade da aplicação, além disso a api_key é uma chave de api gratuita do groq.

Por conta do tempo limitado durante o desenvolvimento, as chaves foram utilizadas diretamente nos testes.

Porém, o projeto pode ser facilmente adaptado para:

* Uso de variáveis de ambiente
* Maior segurança em produção
* Proteção de dados sensíveis

# 🛠️ Notas de Desenvolvimento

## ⚡ Sobre o Comparador de IAs

Devido ao tempo limitado durante o desenvolvimento, a funcionalidade de **Comparar IAs** foi construída de forma mais rápida ("vibe-coded").

Isso significa que:

* A prioridade foi fazer a funcionalidade funcionar corretamente
* O foco foi na entrega dentro do prazo
* Há espaço para refatoração e melhorias futuras

Mesmo assim, a funcionalidade cumpre seu objetivo principal: permitir comparar respostas de diferentes modelos de IA de forma prática.
